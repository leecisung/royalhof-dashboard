# 네이버 검색광고 전환추적 설정 가이드 (TIGER용)

> 작성일: 2026-05-20
> 대상: 로얄호프치킨(4328346) · 버거리(4265143) 계정
> 배경: 두 계정의 ON 캠페인 15개가 전부 `trackingMode=TRACKING_DISABLED` —
> 전환추적이 꺼져 있어 보고서상 "전환 0"이 측정 불능 상태. (진단: `reports/diagnose_20260520.md`)

---

## 0. 핵심 요약

- 네이버 검색광고의 전환수(ccnt)는 **프리미엄 로그분석**을 사이트에 연동해야 집계된다.
- 로얄·버거리 계정은 이 연동이 안 돼 있다. **보승회관(694291) 계정은 이미 돼 있으므로**(캠페인이 `AUTO_TRACKING_MODE`), 그 계정 설정을 참고하면 빠르다.
- 프리미엄 로그분석은 **무료**. 연동하면 그동안의 "전환 0"이 실제 수치로 잡히기 시작한다.
- 이 작업은 API로 대행 불가 — 네이버 UI 로그인 작업이다.

---

## 1. 설정 단계 (계정별로 반복)

### 1-1. 로얄호프치킨 계정 (4328346)

1. **네이버 검색광고 로그인** (searchad.naver.com) — 로얄호프치킨 계정.
2. 상단 **도구 → 프리미엄 로그분석**.
3. **사이트 등록**: `royalhofchicken.com` 등록 (비즈채널에 이미 있으면 선택).
4. **공통 분석 스크립트 설치**: 발급되는 스크립트를 사이트 전체 페이지의 `<head>`에 삽입.
   - GTM(구글 태그 매니저)을 쓰면 GTM에 태그로 추가해도 됨.
   - 사이트 소유확인(메타태그 또는 HTML 파일 업로드)이 필요할 수 있음.
5. **전환 설정**: 창업 페이지의 목표 행동을 전환으로 정의.
   - 창업문의 폼 제출 → "신청·예약" 전환
   - 전화 클릭 → "전화" 전환
   - 전환 페이지 URL(완료/감사 페이지) 기반 또는 이벤트 기반으로 지정.
6. 설치·연동이 끝나면 캠페인 **전환추적이 자동 적용**(`AUTO_TRACKING_MODE`)된다.
   - 자동 적용 안 되는 기존 캠페인은 캠페인 설정에서 전환추적을 켠다.

### 1-2. 버거리 계정 (4265143)

위 1~6을 동일하게 반복. 사이트는 `burgerry.co.kr`.

> 버거리는 Meta Pixel `COMPLETE_REGISTRATION` 이벤트를 이미 쓰고 있다(CLAUDE.md 9장).
> 네이버 전환과 Meta 전환은 별개 측정 채널 — 둘 다 켜 두고 검색 vs SNS 비교에 쓴다.

---

## 2. 보호 캠페인 주의

- `로얄 고비용 (M)·(P)` 등 PROTECTED_CAMPAIGN_IDS 캠페인도 **전환추적은 켜는 게 맞다**.
  전환추적 ON은 성과 측정일 뿐 입찰가·예산을 바꾸지 않으므로 "절대 건드리지 않음" 규칙과 무관하다.
- 단, 보호 캠페인의 `trackingMode`는 자동화/스크립트로 바꾸지 않는다. **네이버 UI에서 직접** 켤 것.

---

## 3. 검증

연동 후 2~3일 데이터가 쌓이면 확인:

```
python scripts/adhoc_query.py campaigns          # trackingMode 확인용은 diagnose 스크립트
python scripts/diagnose_bids_tracking.py         # #3 섹션에서 TRACKING_DISABLED 사라졌는지 확인
python scripts/report_weekly.py --since <월> --until <일>   # 전환수(ccnt) 집계 확인
```

- `diagnose_bids_tracking.py`의 #3 표에서 ON 캠페인의 `TRACKING_DISABLED ⚠️`가 없어지면 성공.
- 주간 보고서의 "전환" 컬럼에 0이 아닌 값이 잡히기 시작하면 정상.

---

## 4. 연동 완료 후 — 자동화로 처리 가능한 것

프리미엄 로그분석 연동이 끝나면, **비(非)보호 캠페인**의 `trackingMode`를 일괄로
`AUTO_TRACKING_MODE`로 전환하는 작업은 API(`PUT /ncc/campaigns/{id}`)로 대행 가능하다.
필요 시 클로드코드에 요청 — 보호 캠페인은 제외하고 처리한다.

## 참고

- GA4: `.env`의 `GA4_PROPERTY_ID`가 비어 있음. GA4는 네이버 ccnt와 별개의 병행 측정 —
  검색광고 전환 자체는 위 프리미엄 로그분석으로 해결되고, GA4는 채널 전반 분석용으로 별도 셋업.
- 메뉴 명칭은 네이버 UI 개편에 따라 다를 수 있음.
